# GROWTH.md — Global Expansion Playbook for OpenOmniBot

> 🎯 **Mission**: Help OpenOmniBot reach 5,000+ GitHub stars by establishing it as the **de facto on-device AI agent standard for Android/Linux power users and developers worldwide**.

## 1. Positioning Strategy

### Current Strengths
- **Unique category**: First open-source on-device AI agent that operates Android/Linux natively
- **Technical depth**: Native Kotlin + Flutter, not a wrapper
- **Active development**: Pushed 2026-04-12, working on English version (issue #106)
- **Rich topics**: agent, ai, android, automation, bot, gui, linux, on-device, skills, alpine

### Global Positioning
**Tagline**: *"Your Android Phone, Controlled by AI — Open Source, Local-First, Privacy-Safe"*

**Target audiences**:
1. 🤖 AI/ML developers building agentic systems
2. 📱 Android power users & ROM developers (custom ROM community)
3. 🔐 Privacy-conscious users who want AI without cloud dependency
4. 🐧 Linux mobile developers (PinePhone, Librem 5 community)

### Competitive Differentiation
| Feature | OpenOmniBot | TaskRabbit | Mobile-Agent | Typical Cloud AI |

## 2. GitHub Stars Growth Strategy

### Phase 1: Technical Community Foundation (Week 1-2)

#### 2.1 README Optimization
- ✅ Already have English README — great!
- Add **star growth badge** at the top: `[![Stars](https://img.shields.io/github/stars/omnimind-ai/OpenOmniBot?style=social)](https://github.com/omnimind-ai/OpenOmniBot)`
- Add a **"Built with OpenOmniBot" showcase section** — community projects using OpenOmniBot
- Add **Contributing section** with beginner-friendly issues labeled `good first issue`

#### 2.2 GitHub Trending Defense
Your 480 stars in 25 days is impressive. To stay on Trending:
- Post a **Show HN** or **Show r/androiddev** when you hit 1,000 ⭐
- Use GitHub's Release feature for every meaningful version (builds + release notes)
- Pin a **"roadmap" Discussion** to drive engagement

#### 2.3 Developer Outreach (High ROI)
| Platform | Action | Template |
|----------|--------|----------|
| r/androiddev | Share demo GIF + one-liner | "OpenOmniBot: AI agent that actually operates your Android phone. Open source, runs locally. Demo in comments." |
| r/sideproject | Share launch story | "Built an AI agent that controls my Android phone — no cloud, no privacy trade-off" |
| r/localllama | Cross-promote | "On-device AI agent for Android with full system control — privacy-first alternative" |
| Hacker News | Show HN | "Show HN: OpenOmniBot — open-source AI agent that operates Android/Linux natively" |

### Phase 2: Viral Demonstration (Week 3-4)

#### 2.4 Demo Video Strategy
Create a **30-second killer demo** showing:
1. "Read my texts and remind me about X"
2. "Open Spotify and play Y"
3. "Schedule a meeting for tomorrow at 3pm"

Share as:
- Twitter/X thread with GIF
- Reddit post with embedded video
- LinkedIn technical post

#### 2.5 Integration Ecosystem
- **MCP Server**: Document and publish an official OpenOmniBot MCP server integration
- **Home Assistant**: Community skill for Android phone as Home Assistant agent
- **Termux integration**: Official Termux + OpenOmniBot workflow guide

### Phase 3: Community Flywheel (Month 2+)

#### 2.6 OpenClaw Skill
OpenClaw is the leading open-source AI agent framework with 100k+ users. Creating an official OpenOmniBot Skill for OpenClaw would:
- Instantly expose OpenOmniBot to OpenClaw's user base
- Enable workflows like "Use OpenOmniBot to control your phone from Claude Code"
- See: https://github.com/Gingiris/gingiris-opensource
