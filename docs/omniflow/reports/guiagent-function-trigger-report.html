触发链路

验收重点是让另一个 GUI agent 从外部项目中触发 OOB 已有 Function，并能把 RunLog 注册为新的 Function。 mock MCP 只替代宿主 OOB MCP，工具名和调用顺序保持与正式合同一致。

01

External Repo

进入 MobileGPT、mobile-use 或 mobile-mcp 的独立目录。

02

Agent Kit

安装 wheel，运行 omniflow-agentkit CLI。

03

MCP Recall

调用 omniflow.recall，返回 decision=hit。

04

Call Function

调用 omniflow.call_function，触发已有 settings_click_path_demo。

05

Ingest RunLog

调用 omniflow.ingest_run_log，注册 install_sample_apk_demo。

06

Call Ingested Function

调用 omniflow.call_function，运行安装型 Function。

注意事项

正式接入面

外部 agent 应优先使用 canonical MCP tools：omniflow.recall、 omniflow.call_function、omniflow.ingest_run_log。 旧 OOB 本地工具名只作为内部兼容/分类层保留，不作为公开 agent toolkit 合同。 Skill 和 docs 负责激活条件与操作约束，不承担可信执行。

Codex CLI Sandbox

read-only 和 workspace-write 会阻断 loopback MCP HTTP。 Codex 外部触发验收使用 danger-full-access，同时 prompt 明确禁止文件修改。
