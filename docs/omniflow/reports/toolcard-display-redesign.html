Agent ToolCard Display Redesign

本次优化只处理 toolcard 的显示投影问题，工具调用和 MCP 执行链路不变。核心改动是把 Chat 与 WebChat 的工具卡合并规则抽成共享投影层，避免 started / progress / completed 事件在 UI 中变成多个卡片、错位或丢失 terminal 输出。

参考项目

  • Vercel AI SDK UI Stream Protocol： 工具调用与工具结果都携带 toolCallId，前端用同一个 ID 把调用生命周期投影成同一个 UI part。
  • OpenHands Event System： 采用 append-only event log，ActionEvent 和 ObservationEvent 分离，UI/服务从事件日志做只读投影。
  • Cline： 将命令执行、浏览器自动化、MCP 工具使用呈现为任务过程的一部分，同时保留用户可追踪的工具输出与审批反馈。

设计原则

不再在各个 UI 分支里临时拼字段，而是统一为：

事件不可变 投影可覆盖 同一工具调用只显示一张卡 terminal delta 追加 错误状态不覆盖输出

落地链路

1. Agent Event

tool_started、tool_progress、tool_completed 继续从端侧流入。

2. Identity

按 toolCallId、callId、cardId、toolTaskId、entryId 建立别名集合。

3. Projection

AgentToolCardProjection 合并已有卡片，追加 terminal delta，保留 args/result/runlog 元数据。

4. Upsert

Chat 与 WebChat 都只消费投影结果，更新同一个 ChatMessageModel。

5. Render

现有 AgentToolSummaryCard 和 activity compactor 继续负责视觉呈现与详情面板。

  • ui/lib/webchat/web_chat_app.dart WebChat 链路改为消费共享投影，修复 terminal delta 不显示。
  • ui/lib/features/home/pages/chat/mixins/agent_stream_handler.dart 旧 mixin 链路同步规则。
  • ui/test/services/agent_tool_card_projection_test.dart 增加投影层单测。

验收点

  • 同一个工具调用的 started / progress / completed 最终只保留一个 toolcard。
  • progress 里的 terminalOutputDelta 会追加到现有 terminal 输出。
  • completed 事件即使换了 entryId，也会合并到占位卡，不再错位成新卡。
  • 工具标题优先级稳定：toolTitle、args 中的 tool_title、display/tool name、summary。
  • 失败状态统一归一为 error，不会把已有 assistant output 覆盖成错误泡。
